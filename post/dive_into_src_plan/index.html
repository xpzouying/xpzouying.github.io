Dive Into Code

之前制定了阅读计划，每天阅读25页书。

想不到自己能够坚持下来，在整个过程中发现如果是阅读纸质书，那么效率高很多，

后来由于各种原因，比如旅游、没有纸质书、上下班的大巴车上等原因，不少书在iPad或者Kindle上阅读，发现效率低了很多。事后自己反省原因，主要有下面几个：

  1. 我自己在iPad/Kindle上面阅读的效率确实没有纸质书上高；

  2. 对于纸质书来说，每天阅读25页书籍，很容易就实现了检查所设定的25页阅读目标（KPI）是否完成，但是对于电子书来说，由于Kindle上面的页码表示与纸质书的页码表示有所差异，所以不容易检查自己设定的KPI是否已经完成；

  3. 前期由于设定自己目标的新鲜热度，所以常常把阅读任务的优先级放在较高的优先级上面，在未完成目标时，有时候会阅读到凌晨直到完成目标才睡觉。后期由于很多时候在Kindle/iPad上面阅读，不知道是否已经完成所设定的目标，并且该目标也属于半放弃的状态，所以导致睡前慢慢开始玩手机，刷各种无脑内容。

总结及规划

总而言之，通过这段时间的目标设定，让自己在这个过程中坚持阅读了不少书，不过大部分都是小说。

现在准备调整目标计划，从小说阅读调整为阅读/解析源码系列。

  1. 坚持每天阅读开源源码；

  2. 坚持每天更新github；
